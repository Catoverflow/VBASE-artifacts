Status milvus::QueryArguments::SetGuaranteeTimestamp ( uint64_t  timestamp)
inline

Instructs server to see insert/delete operations performed before a provided timestamp.
If no such timestamp is specified, the server will wait for the latest operation to finish and query.
.

Note: The timestamp is not an absolute timestamp, it is a hybrid value combined by UTC time and internal flags.
We call it TSO, for more information please refer to:
https://github.com/milvus-io/milvus/blob/master/docs/design_docs/milvus_hybrid_ts_en.md. You can get a TSO from insert/delete results. Use an operation's TSO to set this parameter,
the server will execute query after this operation is finished.
Default value is 1, server executes search immediately.
